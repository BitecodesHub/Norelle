# llm.txt — Norelle Luxury Perfume Platform

## Project Summary
Norelle is a production-ready luxury perfume eCommerce platform built with Next.js 14 (App Router), MongoDB (Mongoose), Razorpay payments, NextAuth.js v5 authentication, Tailwind CSS, Framer Motion, and React Three Fiber for 3D elements.

## Brand Identity
- **Brand Name**: Norelle
- **Tagline**: Scent of Presence
- **Aesthetic**: Minimalist luxury — black (#0E0E0E), soft beige (#E8DCCB), gold (#D4AF37)
- **Typography**: Cormorant Garamond (serif headings) + Inter (sans-serif body)
- **Design Language**: Glassmorphism navbar, claymorphism product cards, GSAP hero animations, 3D perfume bottle (R3F)
- **Target Market**: Luxury fragrance buyers; local SEO focus on Ahmedabad, India

## Technology Stack
- **Framework**: Next.js 16.2.3 (App Router, TypeScript)
- **Database**: MongoDB via Mongoose ODM
- **Auth**: NextAuth.js v5 (beta) — Email/Password + Google OAuth
- **Payments**: Razorpay (order creation + HMAC signature verification)
- **Styling**: Tailwind CSS v4
- **Animations**: Framer Motion + GSAP
- **3D**: React Three Fiber + Drei (glass perfume bottle with gold lighting)
- **State**: Zustand (cart persisted to localStorage)
- **Validation**: Zod (server + client)
- **Image CDN**: Cloudinary
- **Runtime**: Node.js (all API routes), Edge (middleware only, no DB)

## User Roles
| Role     | Permissions |
|----------|-------------|
| USER     | Browse, cart, checkout, orders, account |
| SALESMAN | Create/manage own coupon codes, view coupon analytics |
| ADMIN    | Full access — products CRUD, orders, users, all coupons |

## Key Features
1. **3D Hero**: Black screen → GSAP letter-by-letter reveal → rotating glass perfume bottle (R3F) → tagline fade-in → CTA
2. **Coupon System**: Salesman creates codes (PERCENTAGE or FLAT discount), configures min order, expiry, max uses. Validated server-side on checkout.
3. **Razorpay Integration**: Full flow — create order → open Razorpay modal → HMAC SHA256 verify → mark order PAID → increment coupon usage
4. **Role-Based Middleware**: Edge-safe (no mongoose in middleware) split-config pattern with NextAuth v5
5. **SEO**: Dynamic `generateMetadata`, JSON-LD schemas (Product, LocalBusiness), sitemap.ts, robots.ts, Ahmedabad local SEO pages

## Environment Variables Required
```
MONGODB_URI=                    # MongoDB connection string
NEXTAUTH_SECRET=                # Random secret (openssl rand -base64 32)
NEXTAUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=               # Optional: Google OAuth
GOOGLE_CLIENT_SECRET=           # Optional: Google OAuth
RAZORPAY_KEY_ID=                # Razorpay API key
RAZORPAY_KEY_SECRET=            # Razorpay secret key
CLOUDINARY_URL=                 # Cloudinary for image uploads
```

## Getting Started
```bash
# 1. Fill in .env.local
# 2. Start MongoDB (local or cloud)
cd norelle
npm run dev          # Development
npm run build        # Production build

# Seed sample products:
npm install -D tsx
npx tsx scripts/seed.ts
```
